# Higher-Order Functions

better-py provides a comprehensive set of higher-order functions for functional programming in Python. These utilities help you write more composable, reusable, and readable code.

## Overview

Higher-order functions are functions that:
- **Take functions as arguments** (e.g., `compose`, `pipe`)
- **Return functions** (e.g., `curry`, `flow`)
- **Transform functions** (e.g., `flip`, `partial_right`)

## Available Functions

| Function | Purpose | Module |
|----------|---------|--------|
| **`pipe`** | Left-to-right function chaining | `pipe` |
| **`flow`** | Create reusable function pipelines | `pipe` |
| **`Pipeline`** | Fluent data transformation API | `pipe` |
| **`pipeable`** | Mark functions as pipeable | `pipe` |
| **`compose`** | Right-to-left function composition | `compose` |
| **`compose_left`** | Left-to-right function composition | `compose` |
| **`curry`** | Convert functions to curried form | `curry` |
| **`partial_right`** | Partial application from the right | `curry` |
| **`flip`** | Flip first two arguments | `curry` |

## Quick Examples

### Pipe - Left-to-Right Chaining

```python
from better_py.functions import pipe

def add_one(x):
    return x + 1

def double(x):
    return x * 2

# Chain operations left to right
pipe(5, add_one, double)  # 12
```

### Compose - Right-to-Left Composition

```python
from better_py.functions import compose

add_one = lambda x: x + 1
double = lambda x: x * 2

# Compose functions (right to left)
process = compose(double, add_one)
process(5)  # 12
```

### Curry - Partial Application

```python
from better_py.functions import curry

@curry
def add(x, y, z):
    return x + y + z

# Call with arguments one at a time
add(1)(2)(3)  # 6

# Or multiple at once
add(1, 2)(3)  # 6
```

### Flow - Reusable Pipelines

```python
from better_py.functions import flow

add_one = lambda x: x + 1
double = lambda x: x * 2

# Create a reusable pipeline
process = flow(add_one, double)
process(5)  # 12
```

## Common Patterns

### Data Transformation Pipeline

```python
from better_py.functions import pipe

def fetch_user(user_id):
    return database.query(user_id)

def validate_user(user):
    return user if user.get("active") else None

def get_user_name(user):
    return user.get("name")

# Chain operations
name = pipe(1, fetch_user, validate_user, get_user_name)
```

### Function Composition

```python
from better_py.functions import compose

# Build complex functions from simple ones
increment = lambda x: x + 1
double = lambda x: x * 2
square = lambda x: x ** 2

transform = compose(square, double, increment)
transform(3)  # square(double(increment(3))) = 64
```

### Currying for Reusability

```python
from better_py.functions import curry

@curry
def filter_by(predicate, items):
    return [item for item in items if predicate(item)]

evens = filter_by(lambda x: x % 2 == 0)
evens([1, 2, 3, 4, 5, 6])  # [2, 4, 6]
```

## Choosing the Right Tool

### Use `pipe` when:
- You want to chain operations in a readable way
- Operations follow a natural sequence
- You're processing data through a pipeline

### Use `compose` when:
- You want mathematical composition style
- Building complex transformations
- Creating reusable function combinations

### Use `curry` when:
- You need partial application
- Creating specialized functions from general ones
- Building function factories

### Use `flow` when:
- You want reusable pipelines
- Need to pass the pipeline as a function
- Building transformation utilities
